# iLink App 产品方案与 UI 方向

版本：Hackathon 核心叙事修订 0.3
日期：2026-07-18

## 1. 核心产品定义

**iLink 帮助成年子女与父母记录、整理并主动分享各自的生活，让双方更了解彼此，也更容易知道怎样关心对方。**

英文短句：**Life, shared with care.**

iLink 不是时间管理工具，也不是人生绩效教练。它不要求用户还原完整的一天。它只把用户主动留下的生活片段整理好，并让本人决定哪些部分可以进入家庭关系。

## 2. 核心故事

成年子女和父母之间通常不是没有感情，而是缺少自然、持续的生活上下文：

- 子女不知道父母最近除了身体之外，日常过得怎样。
- 父母只知道子女“工作挺忙”，不知道他正在经历什么。
- 双方每次打电话都从“吃了吗”“最近忙吗”开始，很难进入真正的生活。
- 有些小事不值得专门发消息，却会让对方更了解自己。

iLink 把这些小事变成一个低压力的循环：

```text
随口记录生活
  ↓
AI 每天整理成可修改片段
  ↓
本人确认 AI 是否理解正确
  ↓
本人选择哪些内容可以分享
  ↓
家人收到近况卡并回应
```

价值不在于记录了多少小时，而在于：**今天有没有一件事，让家人比昨天更了解我。**

## 3. 核心用户场景

### 成年子女

用户在外地生活。当天做了产品原型、和朋友讨论方向、买菜时散了会儿步。他不想逐件给父母发消息，也不想晚上重新回忆一遍。

他白天随口留下几段语音。晚上打开“整理”，确认 AI 没有误解，再勾选两条生成给妈妈的近况：

> “今天开始认真做一个新产品，下午也出去走了走。最近挺充实，周日晚上可以打电话。”

### 父母

妈妈当天去跳舞、和朋友吃饭，回家发现厨房水管漏了。她说了两段语音，确认后分享给孩子：

> “今天去跳舞了，和李阿姨吃了午饭。厨房水管有点漏，周六能不能帮我找一位师傅？”

孩子不需要猜怎样关心，可以直接回应：

> “我周六帮你找。”

## 4. App 信息架构

底部只保留三个核心入口：

| 入口 | 用户任务 | 核心内容 |
|---|---|---|
| 家人 | 家人最近主动分享了什么 | 一周/月度家庭简报、家人列表、家人分享给我的近况、具体求助与回应 |
| 记录 | 立刻留下生活，或用语音完成家庭相关动作 | 默认落地页、按住说话、意图确认、记录/分享/查看家人 |
| 我 | AI 有没有正确理解我的一天，以及我分享过什么 | “我的一天”整理、“我的分享”历史、生成家人近况卡 |

“我”占据第三个主导航；硬件与权限管理不占用主导航，用户从“我”或“家人”右上角头像进入：

- iLink Button 硬件配对；
- 与父母的连接和分享权限；
- 原始录音保存周期、数据导出与删除。

## 5. 三个核心页面

### 01. 记录

这是默认落地页，也是最高频操作入口。

- 主视觉只保留一句邀请、声纹和底部按住录音控件。
- 用户可以讲生活片段：“今天开始做新项目。”
- 也可以讲家庭相关意图：“把今天整理成近况给妈妈”“看看爸爸最近怎么样”。
- Agent 先展示识别出的意图、对象、时间范围和权限，再等待确认。
- 普通记录保存后进入“我 → 我的一天”，不会直接分享给任何人。

### 02. 我

这个页面替代旧版“整理”，内部使用两个 Tab：“我的一天”和“我的分享”。它不评价用户，也不展示周/月绩效。

“我的一天”结构：

1. AI 根据当天主动输入生成的一句话生活摘要。
2. 展示“2 / 4 已确认”等明确状态。
3. 每个生活片段包含来源时间、AI 整理结果和原始语义。
4. 用户可编辑或确认每个片段。
5. 全部确认后，用户可以生成家人近况卡。

“我的分享”集中展示用户已经确认并发送给妈妈或爸爸的近况卡，包括接收人、发送时间、已读状态和最终发送内容。草稿、原始录音和未确认片段不出现在这里。

确认的是“AI 有没有理解对”，不是“这件事做得好不好”。

### 03. 家人

第一层默认展示“近期关注”，帮助用户一眼看懂最近一周或一个月的家庭生活：

- 需要跟进：对方明确请求帮助、仍未完成的事项；
- 可以主动帮忙：同一个需求在多张近况卡中反复出现；
- 生活变化：新的习惯、活动或值得下次聊天时主动问起的事情；
- 普通但开心的生活：不需要解决，但能帮助双方更了解彼此。

摘要只能使用家人主动分享给用户的近况卡。每条关注事项必须显示来源，并可以打开对应家人的原始分享记录。没有分享不代表异常，系统不能因为沉默、少记录或单次负面情绪推断风险。

时间范围支持：

- 最近一周：强调需要马上跟进的事情和本周生活变化；
- 最近一月：识别重复出现的需求、新习惯和长期未完成事项。

用户可以切换到“家人列表”，查看妈妈、爸爸、最近分享时间和最新摘要。

点击成员后，只展示该成员主动确认并分享给我的内容。用户发给家人的近况统一进入“我 → 我的分享”，避免在家人页混入自己的输出。

每张卡可以包含：

- 今天或最近发生的事；
- 想让对方知道的感受；
- 一个明确的求助；
- 对方可执行的回应。

家人只能看到经过本人确认发送的近况卡，不能进入对方的原始录音、整理页面或完整生活记录。

## 6. 近况卡编辑器

AI 只能从已确认片段中提议内容。

- 默认勾选低敏感度的生活片段。
- 私密候选默认不分享。
- 用户可以保留、改写或删除每一条。
- 发送前显示接收方最终看到的完整文本。
- 发送必须由本人再次确认。

示例：

```text
已确认片段
  ├─ 开始做 iLink 产品原型      ✓ 默认候选
  ├─ 下午买菜并散步             ✓ 默认候选
  └─ 上午有些焦虑               ○ 默认不分享

最终给妈妈：
“今天开始认真做一个新产品，下午也出去走了走。
最近挺充实，周日晚上可以打电话。”
```

## 7. 语音 Agent 的有限能力

语音 Agent 第一版只支持三个可解释意图：

### 记录生活

把语音整理成一个待确认生活片段。无法确认的信息保持未知，不编造时间、人物或情绪。

### 生成家人近况卡

识别接收人和时间范围，只读取已确认片段，随后进入编辑器。Agent 没有直接发送权限。

### 查看家人信息

识别具体家人，只打开对方主动分享给用户的卡片。不能检索对方的私人记录。

## 8. 隐私与关系边界

1. **默认私密：** 原始语音、转写和“我的一天”只有本人可见。
2. **逐条确认：** AI 整理结果在本人确认前不能用于分享。
3. **分享卡片，不共享数据库：** 家庭连接不会带来原始记录访问权。
4. **逐次发送：** 每张近况卡都需要预览和确认。
5. **成员权限独立：** 给妈妈和给爸爸的内容及连接状态彼此独立。
6. **可删除和导出：** 用户可以管理录音保存周期、导出记录或彻底删除数据。
7. **不推断风险：** 产品不根据少记录、就医或情绪片段自动向家人报警。

## 9. iLink Button 的角色

硬件只降低记录摩擦，不负责理解或分享。

- 长按时才录音，待机绝不录音。
- 松开后通过蓝牙同步到 App。
- 断连时暂存用户主动触发的片段。
- 所有转写、整理、确认和分享都发生在 App。
- Hackathon 只用手机界面模拟 Button 输入，不制作硬件 Demo。

## 10. Hackathon MVP

### 必做

1. 默认打开语音记录页。
2. 按住录音并展示实时声纹。
3. 松开后展示意图与结构化结果供用户确认。
4. 保存后进入“我 → 我的一天”。
5. 支持逐条确认和全部确认。
6. 从已确认内容生成给妈妈的近况卡。
7. 家人页展示妈妈、爸爸，以及他们分别分享给我的内容。
8. 家人页默认展示最近一周关注摘要，并支持切换最近一月。
9. 每条关注事项可以回到具体家人的分享记录。
10. 语音指令可以直接打开分享卡或爸爸详情。
11. “我”页增加“我的分享”，展示已经发送给妈妈或爸爸的近况卡。
12. 头像入口保留硬件配对、家人配对和隐私管理。

### 明确不做

- 不做今天的完整时间线、全天统计或时间管理。
- 不做周回顾、月回顾、90 天目标或成长评分。
- 不做持续监听、定位、被动活动识别或情绪诊断。
- 不让父母查看子女的原始记录，反之亦然。
- 不扩展到情侣、朋友或多人家庭群组。
- 不做社交媒体式动态流、点赞和公开关注关系。

## 11. 90 秒 Demo 主线

1. 用户打开 iLink，默认看到语音记录页。
2. 用户说：“今天开始做 iLink 产品原型，想先把记录、整理和家人分享做通。”
3. Agent 识别为“记录生活”，用户确认加入“我的一天”。
4. “我 → 我的一天”显示 4 个片段，其中 2 个待确认。
5. 用户修改一条、确认全部内容。
6. 用户点击“生成家人近况卡”，保留产品原型和散步，隐藏焦虑片段。
7. 用户预览并发送给妈妈。
8. 切换到家人页，打开妈妈分享给我的卡片：“今天去跳舞了，厨房水管有点漏。”
9. 用户回应：“我周六帮你找师傅。”

收尾句：

> **iLink 不让家人看见你的一切。它让彼此持续看见那些愿意分享、也真正值得关心的生活。**

## 12. UI 方向

继续采用方向 A：**Sunday Letter / 周日来信**。

- 奶油白背景、砖红主色、鼠尾草绿确认状态。
- 中文标题使用带书卷感的宋体，正文使用清晰黑体。
- 记录页克制、安静，突出说话和声纹。
- “我的一天”像编辑一封生活来信，不像任务管理器；“我的分享”像私人寄件箱。
- 家人卡片有明确成员和时间，只展示对方分享给我的内容，不做公开社交动态流。
- 管理能力放入头像入口，不干扰记录、整理和分享的主线。
